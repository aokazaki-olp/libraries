# コーディング規約 (CODING_RULES.md)

> このドキュメントは、プロジェクト全体のコーディング規則およびスタイルを定義するものです。
> 新たな実装やリファクタリングを行う際は、必ず本規約を遵守してください。

---

## 1. 基本思想 (Philosophy)

本プロジェクトのコードは、以下のハイブリッドな思想に基づき記述します。

1. **Javaライクな堅牢性**: 明示的なブロック、厳密なエラーハンドリング、役割の分離。
2. **Modern JS (ES6+) の活用**: GAS V8エンジンを前提とした、簡潔で強力な構文（分割代入、オプショナルチェーン等）の積極採用。

迷った場合は、常に「チームメンバーにとっての**可読性**」と「実行時の**堅牢性**」を優先してください。

---

## 2. モジュール構造とファイル構成

### 2.1 モジュールパターン

各ファイルは役割に応じて明確にパターンを使い分けます。

| パターン | 使用基準 | 使用例 |
|---|---|---|
| **IIFE**<br>`const X = (() => { ... })();` | 内部関数やキャッシュをカプセル化し、公開APIのみを `return` する場合。 | `HttpClient`, `LazyTemplate`, `resolveSheet` 等 |
| **関数宣言**<br>`function name() {}` | 単一機能で完結しており、ホイスティング（定義位置の制約回避）が必要な場合。 | `deepFreeze` 等 |

### 2.2 ファイルヘッダー (JSDoc)

ファイルの役割区分に応じて、冒頭のヘッダーの要否を変えます。
（`@author` や `@version` 等のメタ情報は保守コストになるため廃止（任意）としています。）

1. **アプリケーションコード（メインロジック）**
   - **ファイルヘッダー記載不要**。
   - エントリーポイントとなる外部公開関数に適切な JSDoc を記述することに注力する。
2. **ライブラリ・ユーティリティコード**
   - モジュールとしての独立性・移植性を考慮し、冒頭にシンプルな概要ヘッダーを記載する。
   ```javascript
   'use strict';
   /**
    * {FileName}.gs
    * @description {ライブラリの概要・依存関係など}
    */
   ```

## 3. 命名規則 (Naming Conventions)

### 3.1 変数・定数命名

ES6 以降における性質を踏まえ、**「真の定数」と「再代入不可な変数」で命名を明確に分ける**こと。

| スコープ / 役割 | 命名規則 | 詳細・例 |
|---|---|---|
| **「真の定数」**<br>(静的な設定値) | `UPPER_SNAKE_CASE` | 実行前から固定されている値。<br>例: `const MAX_RETRY_COUNT = 5;` |
| **「定数オブジェクト」** | `UPPER_SNAKE_CASE`<br>+ `Object.freeze` | プロパティの変更も防ぐため必ず凍結する。<br>例: `const CONFIG = Object.freeze({ ... });` |
| **「再代入不可な変数」**<br>(動的生成値) | `camelCase` | 関数内で算出/取得される「上書き防止」のための値。<br>例: `const currentUser = auth.getUser();` |
| 短いスコープ<br>（1〜3行のループ内） | **1文字変数 (推奨)** | 簡約化のため。<br>例: `o`, `k`, `v`, `s`, `n`, `e` |
| 長い・通常のスコープ<br>（コード全体） | **変数名の省略禁止** | オブジェクトのプロパティや変数全般で省略形を使用しない。<br>例: `options` (not `opts`), `config` (not `conf`) |

## 4. 構文・スタイル規則

### 4.1 必須事項 (Java-like Strictness)

| 対象 | ルール | 詳細・理由 |
|---|---|---|
| **ブロックスタイル** | **必須 `{ ... }`<br>+ 改行** | 単一行の `if` や `for` であっても必ず波括弧で囲み、本文は改行して記述する。<br>※ アロー関数の単一式 (`=> expr`) は対象外。<br>❌ `if (x) return;`<br>❌ `if (x) { return; }`<br>✅ `if (x) {`<br>&emsp;&emsp;`return;`<br>`}` |
| `forEach` | **使用禁止** | デバッグ容易性と `break`/`continue`/`await` のため、必ず `for...of` とする。 |
| `var` | **使用禁止** | ブロックスコープ保証のため、`const` または `let` とする。 |
| **`switch` の `default`** | **必須** | 全分岐の網羅を保証するため、必ず `default:` 節を記述する。<br>処理不要の場合もコメントで意図を明示する。<br>✅ `default: // 該当なし` |
| **`switch` の `break`/`return`** | **必須** | フォールスルーを防止するため、各 `case` に必ず `break` または `return` を記述する。<br>意図的なフォールスルーの場合は `// falls through` コメントを付与する。 |
| **Yoda 条件** | **使用禁止** | 可読性を損なうため、定数を左辺に置く比較（`null === x` 等）は禁止する。<br>❌ `if (null === value)`<br>✅ `if (value === null)` |

### 4.2 推奨事項 (Modern JS Features)

GAS V8ランタイムの機能を活用し、簡潔で安全なコードを目指します。

| 対象 | アクション | 例 / 詳細 |
|---|---|---|
| **`==` の使用** | **積極活用** | `== null` による null/undefined の一括チェックなど、意味を理解した意図的な使用を認める。 |
| **存在チェック** | **推奨 (`!obj`)** | `if (!obj)` を使用（但し `0` や `""` が有効値の場合は `===` 等で厳密に）。 |
| **Null合体演算子** | **推奨 (`??`)** | `val ?? default` (`||` と違い `0` や `false` をデフォルト値で上書きしない)。 |
| **オプショナルチェーン** | **推奨 (`?.`)** | `obj?.prop` (Java の Optional 的な安全性確保)。 |
| **分割代入** | **推奨** | `const { id, name } = user;` |
| **デフォルト引数** | **推奨** | `function(options = {})` |
| **スプレッド構文** | **推奨** | `[...arr]`, `{ ...obj }` |
| **アロー関数** | **積極活用** | `x => x + 1` のように引数1つの `()` や、単一式の `{}` `return` を省略する。 |
| **一時変数の排除** | **推奨** | 一度しか利用しない変数は切らず、式に直接埋め込む。 |

## 5. エラー戦略と JSDoc

### 5.1 関数 JSDoc の必須項目
すべてのプロトタイプ/関数には、以下を必ず記述します。
- `@param`
- `@returns`
- `@throws`（例外を送出する場合）
- 設計上の制限事項（該当する場合はブレットリストで特記）

### 5.2 エラーの投げ分け

状況に応じて、送出するエラークラスを使い分けます。

#### [A] 型バリデーションエラー (`TypeError`)
関数冒頭の fail-fast（事前条件）バリデーションなどで型が一致しない場合用。

```javascript
throw new TypeError('{パラメータ名} には {期待型} を指定してください');
```
- **補足**: `{期待型}` は単一の型名に縛られず、人間が読んで理解しやすい自然言語での列挙で構わない。
- 仕様上 `null` や複数の型を許容する場合は、開発者の混乱を防ぐため必ず `'...には Date または null を...'` のように明示すること。

#### [B] ドメインエラー (`Error`)
API通信の失敗、期待するデータ（シート等）が見つからない、など業務ロジック上のエラー用。

```javascript
throw new Error('具体的なドメインエラーメッセージ');
```
